## **2. "USE" 자율 구축 시스템 상세 규격**

### **2.1 과금 시점 확정: "Success-Only" 차감**

* **원칙:** "USE" 쿼터(또는 크레딧)는 **구축 성공 시에만 차감**됩니다.
* **성공의 정의:** 1. AI 로직 생성 완료.
2. Recursive CI 검증 통과 (Zero-Error).
3. 최종 인프라 프로비저닝 완료 신호 수신.
* **실패 시 대응:** 플랫폼 결함 혹은 AI 생성 오류로 구축 실패 시 쿼터는 보존되며, 실패 로그는 **SovereignArchitect**에게 전달되어 자가 학습 데이터로만 소모됩니다. (유저 과금 없음)

### **2.2 추가 크레딧 (Over-Quota)**

구독 쿼터 소진 후 발생하는 추가 요청은 다음 단가를 적용합니다.

* **Additional USE Unit:** **$25.00 / 1회 성공**
* **차감 로직:** 시도 시점에 크레딧을 '홀딩(Hold)'하고, 성공 시 '확정(Capture)', 실패 시 '해제(Release)'하는 트랜잭션 방식을 따릅니다.

## **3. 하이퍼 네이티브 스케줄링 (Latency Shaping)**

커널의 `SovereignScheduler`는 테넌트의 티어에 따라 물리적인 대기 시간을 삽입합니다.

1. **Strict Shaping (FREE/PLUS):** 요청 처리 완료 후 응답 반환 전, 목표 지연 시간까지 `thread::sleep` 또는 비동기 타이머를 강제합니다.
2. **Bypass Mode (PRO/ULTRA):** 커널 가속 모듈(Pingora 기반)이 즉시 응답을 반환합니다.
3. **Real-time Priority (ENTERPRISE):** 커널 내 전용 워커 스레드를 점유하여 컨텍스트 스위칭 오버헤드를 제로화합니다.

## **4. 마켓플레이스 상생 경제 (95:5 & 1% 인센티브)**

* **수익 배분:** 제작자 95%, 플랫폼 5% (고정).
* **AI 채택 보너스:** "USE" 엔진이 마켓플레이스 모듈을 채택하여 성공적으로 구축했을 경우, **플랫폼 수수료 5% 중 1%를 떼어 제작자에게 'AI 적합성 리워드'로 추가 지급**합니다.
* *결과적으로 제작자는 최대 96%의 수익을 가져갈 수 있습니다.*
